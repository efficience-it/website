## Qu'est-ce que JavaScript ?

JavaScript est « un langage de programmation largement utilisé dans le développement web moderne ». Créé à l'origine pour ajouter de l'interactivité aux pages web grâce à la manipulation de contenu et aux animations, il prend désormais en charge le développement côté client et côté serveur, ainsi que les applications mobiles.

Le langage permet la modification dynamique des pages, les réponses aux interactions des utilisateurs et les effets visuels grâce à l'intégration avec HTML et CSS. Les fonctionnalités clés incluent la validation de formulaires, les diaporamas, le chargement asynchrone de données et des fonctionnalités avancées comme les cartes interactives.

Au-delà de l'exécution côté client, JavaScript s'exécute côté serveur via Node.js, permettant aux développeurs de construire des applications évolutives en temps réel, adaptées aux chats en direct, aux tableaux de bord et aux outils collaboratifs.

L'écosystème comprend de nombreuses bibliothèques, frameworks et modules NPM (Node Package Manager) qui accélèrent le développement. React, Angular et Vue.js sont des frameworks simplifiant la création d'interfaces réactives et réutilisables.

## Quelles sont les caractéristiques de chaque framework .js ?

### Node.js

Construit sur le moteur V8 de Google Chrome, cette plateforme open source permet le développement d'applications côté serveur en utilisant JavaScript. Alors que JavaScript cible traditionnellement l'exécution côté client dans le navigateur, Node.js a déplacé ce paradigme vers le « développement de projets côté serveur ».

Son succès provient d'une exécution rapide, d'une facilité d'utilisation et d'un support étendu de bibliothèques. Il est devenu populaire pour les systèmes distribués et les microservices grâce à sa capacité à gérer simultanément de nombreux clients et des volumes de données importants.

### React.js

React.js est une « bibliothèque JavaScript front-end » pour la conception d'interfaces visuelles, développée par Facebook. Elle crée des interfaces interactives et réactives pour les applications monopage (SPA) et des composants UI réutilisables.

L'approche basée sur les composants décompose les interfaces en petits éléments autonomes, chacun représentant des portions spécifiques de l'UI et réutilisables à travers les différentes fenêtres du projet.

Le rendu efficace et réactif est sa principale force. La bibliothèque utilise un « algorithme de rendu virtuel qui détecte les différences » entre l'état actuel de l'UI et les mises à jour, évitant les rafraîchissements complets de l'UI et améliorant significativement les performances.

React s'intègre facilement avec d'autres bibliothèques et frameworks, ce qui explique sa popularité dans le développement web. Il se combine avec des outils gérant la gestion d'état, le routage et le rendu côté serveur.

### Vue.js

Créé par Evan You, Vue.js est un « framework JavaScript open source utilisé pour développer des interfaces utilisateur réactives ». De plus en plus populaire dans le développement web, il adopte l'approche basée sur les composants comme React, avec des composants autonomes contenant leur propre logique et modèles de données.

Reconnu pour sa simplicité et sa facilité d'apprentissage, il est « progressivement adoptable », s'intégrant dans des projets existants sans refonte complète. Il crée des SPA et s'intègre dans des projets plus importants.

Vue.js fournit des « templates », une syntaxe expressive décrivant la structure de l'UI de manière déclarative, combinée avec JavaScript pour des composants dynamiques et réactifs.

Les fonctionnalités avancées incluent la gestion de l'état de l'application via Vuex, la gestion des routes et les capacités de rendu côté serveur.

### Angular

Le framework open source de Google permet de concevoir des « applications web dynamiques, robustes et évolutives ». Basé sur TypeScript, une version typée de JavaScript, il offre des avantages en termes de maintenabilité du code, de productivité et de qualité.

Les concepts clés incluent :

- **Approche basée sur les composants** avec des éléments UI réutilisables comprenant des templates HTML, une logique de contrôleur et du style CSS
- **Data binding bidirectionnel** entre les composants et les vues, simplifiant la gestion des données et offrant des expériences réactives
- **Système d'injection de dépendances** pour la modularité et la testabilité
- **Module de routage** pour la navigation entre les vues de l'application avec une gestion propre des paramètres
- **Services** en tant que classes réutilisables fournissant des fonctionnalités spécifiques comme l'accès aux données distantes et le stockage local, injectés dans les composants pour le partage de fonctionnalités et de données

Angular offre « un framework puissant et complet pour le développement web » avec de nombreuses fonctionnalités avancées permettant une architecture modulaire et une haute productivité pour les projets à grande échelle.

## Quel framework JavaScript choisir ?

Comme pour la sélection d'un éditeur de code pour les projets Symfony, le choix du framework dépend de plusieurs facteurs : les exigences du projet, sa taille, les compétences de l'équipe et les préférences.

Il n'existe pas de réponse unique — il faut considérer l'environnement du projet, les attentes du client et les capacités des développeurs. Chaque framework présente des avantages et des inconvénients distincts nécessitant une évaluation attentive pour un ajustement optimal au projet.
